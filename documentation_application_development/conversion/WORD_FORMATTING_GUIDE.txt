WORD FORMATTING GUIDE
CS6004ES Application Development Coursework 2

Recommended final document structure:

1. Title page
2. Abstract
3. Table of contents
4. Introduction
5. Requirements mapping
6. System architecture
7. Database design
8. Feature implementation
9. Validation, security and exception handling
10. Testing and evaluation
11. Individual contribution
12. Conclusion
13. References
14. Appendices

Suggested formatting:

- Paper size: A4
- Font: Aptos, Calibri or Times New Roman
- Body font size: 11 or 12 pt
- Line spacing: 1.15 or 1.5, depending on coursework guidance
- Headings: use Word heading styles for automatic contents
- Figure captions: Figure 1, Figure 2, etc.
- Table captions: Table 1, Table 2, etc.
- Code references: use monospace font
- Appendix labels: Appendix A, Appendix B, etc.

Before export:

- Replace every [NEEDS CONFIRMATION] marker or move it into limitations.
- Check that screenshots and diagram numbers match the report text.
- Redact all sensitive configuration values.
- Confirm the GitHub/source link is accessible.
- Export the final document to PDF if required.
