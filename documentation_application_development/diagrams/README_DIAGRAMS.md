## Export Instructions

These files use Mermaid syntax and can be exported as PNG or PDF using one of the following methods:

1. Open the `.mmd` file in a Mermaid-compatible editor, such as Mermaid Live Editor or a Visual Studio Code Mermaid preview extension.
2. Export each diagram as PNG for screenshots or PDF for high-resolution report insertion.
3. Keep the exported file names aligned with the source names, for example `architecture_diagram.png`.
4. Place exported image files in this same folder or in the report's final image folder.
5. Insert each exported diagram in the report using the figure numbers listed above.

## Verification Notes

- The architecture, class and ER diagrams use actual scanned project names where available.
- `StudentController`, `LecturerController`, `AdminController` and a separate `Grade` model were not detected in the scanned repository. The class diagram marks these expected coursework terms as `[VERIFY WITH CODE]`.
- Reports are generated by `ReportsController` from database queries; no persisted `Report` entity was detected.
- Student, Lecturer and Administrator are Identity roles, not separate entity model classes in the scanned code.
